OLT ADD / EDIT FORM

Fields:

Required:

- OLT Name
- Vendor
- IP Address
- SNMP Version (v1 / v2c / v3)
- SNMP Community
- SNMP Port

Optional:

- Username
- Password
- SSH Port
- Telnet Port
- Notes

Port Defaults:

- Pre-fill SNMP Port = 161
- Pre-fill SSH Port = 22
- Pre-fill Telnet Port = 23

IMPORTANT:

These are only default values.

Users must be allowed to change them.

Never assume ports are always 161/22/23.

IP Address field:

Allowed:

Valid:
192.168.1.1

Invalid:

http://192.168.1.1
https://192.168.1.1

Ports must never be entered inside the IP field.

Ports must use dedicated port fields.

FRONTEND VALIDATION

Before Save:

Validate:

- OLT Name required
- Vendor required
- Valid IPv4 required
- SNMP Version required
- SNMP Community required
- SNMP Port required
- SNMP Port range 1–65535

==================================================
REAL OLT TEST CONNECTION

Create:

POST /api/olts/test-connection

Input Example:

{
"ip": "103.111.225.76",
"snmpVersion": "v2c",
"snmpCommunity": "public",
"snmpPort": 40109
}

Behavior:

Perform SAFE READ-ONLY SNMP VALIDATION ONLY.

Allowed OIDs:

- sysName
- sysDescr
- sysUpTime

Forbidden:

- SNMP SET
- ONU Reboot
- ONU Disable
- ONU Enable
- Configuration Changes
- Write Operations
- Bulk SNMP Walk
- Aggressive Queries

Safety:

- Timeout = 3 seconds
- Retry = 1
- CPU-safe
- No background polling
- No continuous retries

Success Response:

{
"success": true,
"vendor": "...",
"model": "...",
"firmware": "...",
"systemName": "...",
"latencyMs": 25
}

Failure Response:

{
"success": false,
"message": "OLT connection failed. Check IP, SNMP version, community and port."
}
